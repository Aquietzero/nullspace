# 概念关系图生成

为指定的数学文章生成概念关系图，并添加到文章底部。

## 输入

$ARGUMENTS — 文章的文件路径（相对于项目根目录或绝对路径）

## 执行流程

### 1. 读取文章

读取 `$ARGUMENTS` 指定的文章全文，理解文章的核心概念、定义、定理、性质、例子及其相互关系。

### 2. 提取概念并构建关系

从文章中提取 **核心概念**，按以下原则分组：

- **蓝色 (blue)**：核心定义和基础概念（主要定义、公理条件等）
- **橙色 (orange)**：性质和定理（定理、推论、重要性质）
- **绿色 (green)**：具体例子和实例
- **紫色 (purple)**：方法和工具（判定方法、构造方法等）
- **红色 (red)**：特殊/重要概念（需要特别强调的内容）
- **灰色 (gray)**：辅助概念或补充说明

构建概念之间的关系：
- 用 **实线** 表示直接关系（包含、满足、具有等）
- 用 **虚线 (dashed)** 表示跨组联系或类比关系

### 3. 设计布局坐标

为每个节点分配 x, y 坐标，遵循以下原则：

- **核心概念** 放在图的中心偏上位置（约 y=300-400）
- **条件/定义** 放在上方（y=0-200）
- **性质/定理** 放在左下方
- **例子** 放在右下方
- 节点之间保持足够间距（至少 100px），避免重叠
- 整体布局美观，有清晰的层次感和视觉流向
- 画布大约在 x: -200~800, y: 0~650 范围内

### 4. 生成 JSON 文件

生成 JSON 文件，保存到 `source/assets/data/graphs/` 目录。文件名从文章内容推导，使用小写英文和短横线（如 `normal-subgroup-quotient.json`）。

JSON 格式：

```json
{
  "title": "图表标题（中文）",
  "options": {
    "nodeRadius": 30,
    "fullWidth": true
  },
  "nodes": [
    {
      "id": "唯一标识（英文）",
      "label": "显示文本",
      "group": "blue|orange|green|purple|red|gray",
      "shape": "circle|rect|diamond",  // rect 用于核心/分类节点
      "note": "注释文字，支持 $LaTeX$",
      "x": 数字,
      "y": 数字,
      "radius": 数字  // 可选，默认30，重要节点可以更大（如38）
    }
  ],
  "edges": [
    {
      "source": "起点节点id",
      "target": "终点节点id",
      "label": "关系标签（可选）",
      "style": "solid|dashed"  // 可选，默认solid
    }
  ]
}
```

节点设计要点：
- 核心概念和分类节点使用 `"shape": "rect"`
- 普通概念使用默认圆形（不设置 shape）
- `note` 字段用于放 LaTeX 公式或简短说明
- 节点数量控制在 **12-20 个**
- 每个颜色分组代表一个主题聚类

### 5. 更新文章

检查文章底部（`# 小结` 之后）是否已有概念图：

- **如果已有** `<div class="concept-graph" ...></div>`：更新其 `data-src` 属性指向新的 JSON 文件
- **如果没有**：在文章末尾添加以下内容：

```markdown
# 概念关系图

下图展示了本篇的核心概念及其相互关系：

<div class="concept-graph" data-src="/assets/data/graphs/文件名.json"></div>
```

### 6. 汇报结果

告诉用户：
1. 生成的 JSON 文件路径
2. 概念图中包含了哪些核心概念（按分组列出）
3. 如需调整布局或概念，可以直接编辑 JSON 文件
